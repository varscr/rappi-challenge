# Technical Post-Mortem: Rappi Competitive Intelligence System

## 1. Project Objective
Build a robust, automated system to collect competitive data (Rappi, Uber Eats, DiDi Food) across 20 strategic addresses in Mexico and generate actionable strategic insights.

## 2. The Tech Stack (Elite Selection)
*   **Scrapling**: Chosen for its adaptive selectors and built-in stealth engines.
*   **Playwright (DynamicSession)**: Used to handle Single Page Application (SPA) rendering and JavaScript-heavy interactions.
*   **curl_cffi**: Utilized for low-level TLS/JA3 fingerprinting to mimic real browser handshakes during API calls.
*   **Pydantic**: Acts as the "Data Contract" to ensure 100% schema integrity.
*   **Streamlit & Folium**: Built for executive-level visualization and geospatial analysis.

---

## 3. Development Journey: Methods & Attempts

### Attempt 1: "Human-in-the-Loop" UI Automation
*   **Method**: Simulate clicks on the address bar, type addresses, and click suggestions.
*   **Result**: **BLOCKED.**
*   **Blocker**: Rappi and DiDi use transparent overlays (DataDome) that "ghost" clicks from headless browsers. Playwright timed out waiting for elements that were visible to humans but hidden from bots.

### Attempt 2: Internal API Reverse Engineering
*   **Method**: Intercept network traffic to find JSON endpoints like `/api/search/v2/search`.
*   **Result**: **404/403 ERROR.**
*   **Blocker**: API obfuscation. The servers return 404 even for valid URLs if a specific `x-authorization` Bearer token (generated by JS challenges) or a correlated `deviceid` is missing.

### Attempt 3: State Injection (The "Teleport" Strategy) 🏆
*   **Method**: Bypassing the UI entirely by injecting Base64-encoded location cookies (`currentLocation`) directly into the browser context.
*   **Result**: **SUCCESS.** 
*   **Outcome**: This allowed the scraper to "teleport" into a specific zone (e.g., Polanco) without clicking a single button, landing directly on the data pages.

### Attempt 4: Multi-Stage Extraction
*   **Method**: Parsing the `__NEXT_DATA__` JSON block embedded in the HTML.
*   **Result**: **HIGH QUALITY.**
*   **Outcome**: By reading the raw Next.js state, we avoided fragile CSS selectors that change during A/B testing.

| Platform | Primary Blocker | Resilience Solution |
| :--- | :--- | :--- |

| **Uber Eats** | Heavy Redirects | Session Persistence + Dynamic Fetching |
| **DiDi Food** | Login Wall Redirect | Ticket Cookie Auth + API Interception |

## 5. Data Integrity
All data is collected via real-time scraping. No synthetic or estimated data is used. The system gracefully degrades when a platform blocks a request — it logs the failure and continues with the remaining platforms/addresses.
